# 戶外/救災情境無線電通聯指南

> 隊伍分工、集合點規劃、失聯處理、電量管理、備援通訊完整 SOP

---

## 30 秒摘要

1. **通訊是團隊安全的命脈**：出發前務必測試、頻道一致、電量充足
2. **三層通訊架構**：主頻道（全隊）、備用頻道（主頻故障）、緊急頻道（SOS）
3. **失聯處理口訣**：三呼叫 → 換頻道 → 高功率 → 派人找 → 報案
4. **電量管理**：出發 100%、中途 50%+、返程 30%+、保留備用電池
5. **備援優先順序**：手機 112/119 → 衛星通訊 → 哨子三短聲 → 鏡子反光

---

## 適用情境 / 不適用情境

### ✅ 適用情境
- 登山健行隊伍通訊
- 露營活動團隊協調
- 救災志工現場通聯
- 車隊/機車隊編隊行進
- 大型活動現場工作人員

### ❌ 不適用情境
- 單獨行動（無通訊對象）
- 需保密通訊（本指南以開放頻道為主）
- 海上/航空通訊（需專用規範）

## 風險優先原則

| 優先順序 | 通訊風險 | 處置原則 |
|---------|---------|---------|
| 1 | 人員失聯（可能遇險） | 立即啟動失聯處理流程 |
| 2 | 通訊完全中斷 | 依約定集合點集合、啟動備援通訊 |
| 3 | 電量不足 | 低功率模式、減少通話、分配剩餘電量 |
| 4 | 單一頻道干擾 | 切換備用頻道 |
| 5 | 設備故障 | 使用備用機或併隊通訊 |

## 頻道規劃

### 三頻道架構

| 頻道類型 | 用途 | 何時使用 |
|---------|------|---------|
| **主頻道** | 全隊日常通聯 | 正常行進、一般回報 |
| **備用頻道** | 主頻道故障/干擾 | 主頻道無法使用時 |
| **緊急頻道** | 僅限緊急呼叫 | 人員受傷、失聯、請求支援 |

### 頻道切換時機

1. **主頻道嚴重干擾** → 指揮宣布切換備用頻道
2. **緊急狀況發生** → 優先使用緊急頻道，避免佔用主頻
3. **分隊獨立行動** → 小隊使用分配頻道，定時回主頻回報

## 現場流程 SOP

### 步驟 1：集合點通訊測試

**目標**：確認所有人通訊正常

**流程**：
1. 指揮呼叫：「各單位注意，這裡是 XX 隊一號，通訊測試，聽到請依序回報。」
2. 各單位依序回報：「XX 隊二號收到」「XX 隊三號收到」...
3. 確認全員收到後：「通訊測試完成，出發。」

**未回應處理**：
- 再呼叫該單位 2 次
- 確認設備設定
- 必要時更換頻道或設備

---

### 步驟 2：定時回報機制

**目標**：掌握隊伍動態、及早發現異常

**回報頻率建議**：

| 情境 | 回報間隔 |
|------|---------|
| 危險地形（斷崖、溪谷） | 每 15-30 分鐘 |
| 視線不佳（霧、密林） | 每 30 分鐘 |
| 一般行進 | 每 1 小時 |
| 輕鬆路線 | 每 2 小時 |
| **到達重要節點** | 立即回報 |

**回報內容（LACES）**：

| 項目 | 內容 |
|------|------|
| **L**ocation（位置） | 目前所在位置 |
| **A**ctivity（活動） | 正在做什麼 |
| **C**ondition（狀況） | 人員狀態、有無問題 |
| **E**TA（預計時間） | 預計抵達下一點時間 |
| **S**pecial（特殊） | 需要支援或注意事項 |

**範例**：
```
「XX隊一號，這裡是XX隊三號。
位置：三叉路口涼亭。
狀況：全員到齊，無異常。
預計 30 分鐘後抵達稜線。完畢。」
```

---

### 步驟 3：集合點設定

**目標**：失散時有明確集結位置

**設定原則**：
1. 選擇**明顯地標**（涼亭、三叉路、山屋）
2. 至少設定 **2 個集合點**（前段、後段各一）
3. 約定**等待時間**（如 30 分鐘無人到則往下一集合點）
4. 有**備選方案**（若主集合點無法使用）

**集合點資訊格式**：

| 編號 | 名稱 | 位置描述 | 座標（若有）| 等待時間 |
|------|------|---------|-----------|---------|
| 1 | 登山口涼亭 | 步道入口左側 | - | 30 分鐘 |
| 2 | 三叉路鐵牌 | 2.3K 處叉路 | - | 30 分鐘 |

### 步驟 4：失聯處理流程

**目標**：儘速找回失聯者或確認情況

```
┌─────────────────────────────────────────────────────────┐
│                    發現失聯                              │
│           （超過回報時間 / 呼叫無回應）                   │
└──────────────────────┬──────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────┐
│ 步驟 1：主頻道呼叫 3 次                                  │
│        每次間隔 1 分鐘，清楚說明呼叫對象                 │
└──────────────────────┬──────────────────────────────────┘
                       ▼ 無回應
┌─────────────────────────────────────────────────────────┐
│ 步驟 2：切換備用頻道，再呼叫 3 次                        │
└──────────────────────┬──────────────────────────────────┘
                       ▼ 無回應
┌─────────────────────────────────────────────────────────┐
│ 步驟 3：切換高功率（H），在空曠處再呼叫                  │
└──────────────────────┬──────────────────────────────────┘
                       ▼ 仍無回應
┌─────────────────────────────────────────────────────────┐
│ 步驟 4：派 2 人前往失聯者最後已知位置                    │
│        搜索者須保持與指揮通訊                           │
└──────────────────────┬──────────────────────────────────┘
                       ▼ 30 分鐘內未尋獲

└─────────────────────────────────────────────────────────┘
```

**呼叫範例**：
```
「XX隊四號、XX隊四號、XX隊四號，這裡是XX隊一號。
你已超過回報時間，聽到請立即回答。完畢。」
```

### 步驟 5：電量管理

**目標**：確保關鍵時刻有電可用

**電量分配原則**：

| 階段 | 建議剩餘電量 | 策略 |
|------|-------------|------|
| 出發時 | 100% | 全滿出發 |
| 中途（行程一半） | ≥ 50% | 開始節電 |
| 返程開始 | ≥ 30% | 僅必要通訊 |
| 返程結束 | ≥ 10% | 備用電池待命 |
| **備用電池** | 100%（全程保留） | 緊急使用 |

**省電技巧**：

1. **平時使用低功率（L）**：近距離足夠
2. **縮短通話時間**：說完即放開 PTT
3. **減少掃描功能**：掃描非常耗電
4. **非值班者可關機**：輪流監聽
5. **關閉不需要的功能**：背光、鍵盤音

**電量不足時**：
- 通知指揮：「XX隊三號電量低於 20%，改為接收待命。」
- 併入鄰近隊員通訊
- 使用備用電池

---

### 步驟 6：緊急狀況通報

**目標**：快速傳遞關鍵資訊

**緊急呼叫格式**：
```
「緊急呼叫、緊急呼叫、緊急呼叫！
這裡是 XX隊三號。
地點：[位置描述/座標]
狀況：[簡述緊急情況]
需求：[需要什麼支援]
完畢。」
```

**範例**：
```
「緊急呼叫、緊急呼叫！這裡是XX隊三號。
位置：2.5K 處大石旁。
狀況：隊員扭傷腳踝，無法行走。
需求：需要擔架支援。完畢。」
```

**指揮回應**：
```
「XX隊三號，一號收到緊急呼叫。
維持待命，派員前往。請保持頻道暢通。」
```

## 備援通訊方案

當無線電無法聯繫時，依序使用：

| 優先順序 | 方法 | 說明 |
|---------|------|------|
| 1 | **手機 112** | 有訊號時最有效，可跨電信業者 |
| 2 | **手機 119** | 直接聯繫消防救援 |
| 3 | **衛星通訊器** | 無訊號區保命設備（如 Garmin inReach） |
| 4 | **哨子三短聲** | 國際山難求救訊號，重複吹 |
| 5 | **鏡子反光** | 日間對直升機或遠方 |
| 6 | **鮮豔衣物揮動** | 增加目視發現機率 |
| 7 | **派人傳話** | 最後手段，選派體能佳者 |

## 常見錯誤與後果

| 常見錯誤 | 後果 |
|---------|------|
| 1. 出發前沒做通訊測試 | 行進中才發現設定不同，無法通聯 |
| 2. 沒有約定備用頻道 | 主頻道干擾時全隊失聯 |
| 3. 沒有分配呼號 | 呼叫混亂，不知道誰在通話 |
| 4. 回報不規律 | 失聯時無法判斷最後位置 |
| 5. 忽略電量管理 | 緊急時刻沒電可用 |
| 6. 長時間佔用頻道 | 其他人無法通報緊急狀況 |
| 7. 沒有指定集合點 | 失散後不知往哪集合 |
| 8. 只靠無線電無備援 | 設備故障時完全失聯 |
| 9. 發現失聯不處理 | 延誤救援黃金時間 |
| 10. 緊急呼叫不說位置 | 搜救無從下手 |

## 緊急狀況處置

### 🆘 人員受傷無法移動

1. 立即通報指揮，說明位置與傷況
2. 確保傷者安全（遮蔽、保暖）
3. 有急救能力者留守照護
4. 派人接應救援或擔架
5. 持續通訊更新狀況

### 🌧️ 天氣惡化需撤退

1. 指揮評估是否全隊撤退
2. 通報所有單位：「全體注意，天氣惡化，準備撤退。」
3. 清點人數、確認位置
4. 依序撤離，押隊確認無人落後
5. 抵達安全處後回報

### 📵 通訊全面中斷

1. 往約定集合點移動
2. 使用備援通訊方式（手機、哨子）
3. 視野開闘處等待
4. 若有危險優先自保
5. 做出明顯標記便於搜尋

## ❓ FAQ

**Q1：多久沒回報算失聯？**
> 超過約定回報時間 15 分鐘未回應，應啟動失聯處理流程。

**Q2：失聯後多久該報警？**
> 內部搜尋 30 分鐘無結果，或判斷情況可能嚴重，立即撥打 119/112。

**Q3：電量剩多少該換備用電池？**
> 剩餘 20% 時應切換備用電池，原電池留作最後備援。

**Q4：緊急呼叫可以打斷其他通話嗎？**
> 可以。緊急呼叫優先，所有非緊急通訊應暫停。

**Q5：分隊行動時如何通訊？**
> 各隊使用分配的頻道對內通聯，約定時間於主頻道回報指揮。

**Q6：押隊的通訊責任是什麼？**
> 確認無人落後、回報隊尾狀況、協助落隊者通訊。

**Q7：如果只有一台無線電故障怎麼辦？**
> 讓該人員緊跟持有正常設備的隊友，透過他轉達。

**Q8：夜間可以減少通訊嗎？**
> 已紮營安全後可減少，但指揮或值班者應保持監聽。
