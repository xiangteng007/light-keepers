何謂ATOM feed？
ATOM feed是以XML表示CAP檔案下載位置的目錄。每個後面的連結網址是一個ATOM feed，各代表一種CAP分類。
顯示的時間規則為？
每個ATOM feed顯示最新一則CAP時間起回溯七天內的所有CAP連結，如最新一筆資料是在2015年9月9日發布，在ATOM feed連結中即會顯示自9月3日至9月9日這段期間的所發布的所有示警。詳細規則圖例說明
如何選擇適合的取得方式？
  • 若您使用的目的是希望下載到最新的cap檔案，可以撰寫程式藉由ATOM feed中所提供的固定連結依時間序取得每一個CAP檔案的連結，本站每分鐘會更新ATOM feed資料, 因此可以持續取得新的CAP檔。程式亦可參考：程式讀取atomfeed下載示警資料範例
  • 若您只希望取得最新的示警相關訊息，也可以使用RSS閱讀器，直接讀取ATOM feed連結網址(請使用所有示警的連結)，如此可以接收最新CAP示警訊息。
  • 若是網站希望引用示警的資料，建議使用生效中的示警，並週期性抓取連結的更新，如此可不必自行解析每個示警中的生效期間。
  • RSS閱讀器：
    • 電腦推薦使用Internet Expore(IE)或 Firefox 瀏覽器開啟Atom feed 連結。
    • 手機則建議於 APP Store 搜尋「RSS Reader」挑選合適之APP使用。網路上之RSS推薦訂閱工具

何謂JSON？
  • JSON（JavaScript Object Notation）是一種由道格拉斯·克羅克福特構想設計、輕量級的資料交換語言，以文字為基礎，且易於讓人閱讀。儘管JSON是Javascript的一個子集，但JSON是獨立於語言的文字格式，並且採用了類似於C語言家族的一些習慣。
  • JSON 資料格式與語言無關，脫胎於 JavaScript，但目前很多程式語言都支援 JSON 格式資料的生成和解析。JSON 的官方 MIME 類型是 application/json，副檔名是 .json。(Reference :WIKI)
顯示的時間規則為？
每個JSON顯示最新一則CAP時間起回溯七天內的所有CAP連結，如最新一筆資料是在2015年9月9日發布，在JSON 連結中即會顯示自9月3日至9月9日這段期間的所發布的所有示警。
如何選擇適合的取得方式？
  • 若您使用的目的是希望下載到最新的cap檔案，可以撰寫程式藉由JSON 中所提供的固定連結依時間序取得每一個CAP檔案的連結，本站每分鐘會更新JSON 資料, 因此可以持續取得新的CAP檔
  • 若是網站希望引用示警的資料，建議使用生效中的示警，並週期性抓取連結的更新，如此可不必自行解析每個示警中的生效期間。
